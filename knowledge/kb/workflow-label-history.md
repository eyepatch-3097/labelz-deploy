# Label History in Labelz (Batches, Reprints, and Audit)

**Label History** helps you track what has already been generated in Labelz—so you can **reprint**, troubleshoot, and maintain an audit trail of label output over time.

This is especially useful when:
- a print run gets damaged and you need to reprint the exact same labels
- you want to confirm which template was used for a past batch
- you want to track label generation activity inside your Org

---

## What Label History typically includes

Depending on your plan and permissions, Label History can show:

### 1) Single label generations
- labels generated one-off from a template
- useful for quick reprints and testing

### 2) Bulk batches (Bulk Generation runs)
For each batch, you can usually see:
- batch name/identifier (friendly)
- template used
- date/time created
- number of labels generated
- status (completed / failed if applicable)
- export/output file link (if available)

---

## How to use Label History (common workflows)

### A) Reprint a previous batch
1. Go to **Label History** (or Batches/History section)
2. Search or filter by:
   - template name
   - workspace
   - date range
3. Open the batch
4. Download the existing output again (recommended)
5. Print at actual size

✅ Tip: Reprinting from history avoids accidental changes that can happen if you regenerate with a modified template.

---

### B) Confirm which template was used
This is useful if you have multiple similar templates.

1. Open the batch from history
2. Look for the “Template used” reference
3. If the label output is off, compare:
- template design version
- field values used
- print settings

---

### C) Troubleshoot failures or unexpected output
When something looks wrong:
- check the batch preview/output in history
- confirm the input data and field formats
- verify whether the template was edited after the batch was created

**Common causes:**
- template changed after last successful print
- inconsistent formats in bulk input (dates, currency)
- printer scaling differences

---

## Best practices for reliable history and reprints

### Name your batches clearly
Use names like:
- `Retail Tags - Kurta Drop - Mar 2026 - 500 pcs`
- `Amazon Box Labels - Restock - 1200 pcs`
- `Cosmetics Stickers - Shade Set A - Feb 2026`

Clear naming makes history searchable and reduces confusion.

---

## Access control and history visibility
Label History is typically permission-based.

If you can’t see History or can’t access a batch:
- you may not have permission in the Org/workspace
- ask an Org admin to grant access

---

## Frequently asked questions

### Does history change if I edit a template later?
Usually, the batch record remains, but if you regenerate labels later using the updated template, the output may change.

**Best practice:** for reprints, use the **existing batch output** instead of regenerating.

### Can I delete history?
No, you cannot delete label templates generated history.
