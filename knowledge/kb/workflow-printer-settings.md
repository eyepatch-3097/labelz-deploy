# Workflow: Printer Settings for Clean Label Printing

Printing is where most label issues show up—cut-offs, misalignment, tiny text, or barcodes that don’t scan.

This guide helps you set the most reliable printer settings for **Labelz outputs** so labels print cleanly and consistently.

---

## Before you print (quick checklist)
- [ ] Confirm the **label size** (your sticker/tag dimensions)
- [ ] Confirm the export format (usually **PDF**)
- [ ] Print **one test page** first (always)
- [ ] If using barcode/QR: plan a quick **scan test** after printing

---

## Step-by-step: Recommended print settings (most reliable)

### Step 1 — Use the correct paper/label size
In your print dialog:
1. Choose the correct **Paper Size**
   - If you are printing on A4 sticker sheets: select **A4**
   - If you are printing tags on a specific sheet: select that exact size if available
2. Confirm **Orientation**
   - Portrait vs Landscape based on the exported layout

✅ Best practice: Use the exact label sheet size (A4/Letter/custom) and align your design to that.

---

### Step 2 — Set scaling to “Actual Size”
This is the #1 reason labels get cut off.

In the print dialog (or PDF viewer print settings), set scaling to:
- **Actual Size** (preferred)
- **100%**
- Disable:
  - “Fit to page”
  - “Shrink oversized pages”
  - “Scale to fit”

✅ If you must use fit-to-page, then the template design must be built specifically for that behavior.

---

### Step 3 — Set margins / borderless printing correctly
If your label is near the edge:
- Use **Normal margins** unless you have a borderless printer
- If you have a printer that supports borderless:
  - enable **Borderless** mode (if needed)
  - confirm it doesn’t auto-scale the page

✅ Best practice: In template design, keep important content away from the edges to avoid dependence on borderless printing.

---

### Step 4 — Print quality settings (important for barcode/QR)
If you have barcode/QR on your label:
- Set print quality to:
  - **High** / **Best**
  - Avoid “Draft” mode

✅ Low quality printing can make codes unscannable even if the design is correct.

---

### Step 5 — Do a one-page test print + scan test
1. Print one page (or one label sample)
2. Check:
   - cut-off edges
   - readability
   - alignment on sticker sheet
3. If barcode/QR exists, scan it:
   - scan 3–5 samples from different positions on the page

---

## Common printing problems and fixes

### “Label is getting cut off”
**Likely cause:** scaling or margins  
**Fix:**
- set scaling to **Actual Size / 100%**
- increase margins/padding in template design
- ensure correct paper size selection

---

### “Everything looks smaller than expected”
**Likely cause:** “fit to page” shrinkage or incorrect page size  
**Fix:**
- print at **100%**
- confirm you selected A4 vs Letter correctly

---

### “Sticker sheet alignment is off”
**Likely cause:** incorrect label sheet template or printer feed differences  
**Fix:**
- confirm the exact label sheet dimensions
- do a “calibration” test:
  - print on plain paper
  - place behind sticker sheet against a light source
  - adjust margins in design if needed

---

### “Barcode/QR doesn’t scan”
**Likely cause:** small code size, low quality print, or insufficient whitespace  
**Fix:**
- increase code size in Template Design
- add more whitespace (quiet zone)
- increase print quality to High/Best
